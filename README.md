## ğŸš€ Deployment to GitHub Pages

### Step 1: Create GitHub Repository
1. Go to [GitHub](https://github.com) and create a new repository
2. Name it `your-portfolio` or similar
3. Make it public (required for free GitHub Pages)
4. Don't initialize with README (since we already have files)

### Step 2: Upload Files
```bash
# Navigate to your project folder
cd portfolio-website

# Initialize git repository
git init

# Add remote origin (replace with your repository URL)
git remote add origin https://github.com/your-username/your-repo.git

# Add all files
git add .

# Commit changes
git commit -m "Initial commit: Portfolio website"

# Push to GitHub
git push -u origin main
```

### Step 3: Enable GitHub Pages
1. Go to your repository on GitHub
2. Click **Settings** tab
3. Scroll down to **Pages** section
4. Under **Source**, select **Deploy from a branch**
5. Choose **main** branch and **/ (root)** folder
6. Click **Save**

### Step 4: Access Your Website
Your portfolio will be available at:
`https://your-username.github.io/your-repo-name/`

## ğŸ› Troubleshooting

### Common Issues

**Images not loading:**
- Check file paths in HTML
- Ensure images are in the correct folder
- Verify file names match exactly (case-sensitive)

**Animations not working:**
- Check if AOS library is loaded
- Verify internet connection for CDN resources
- Check browser console for JavaScript errors

**Mobile menu not working:**
- Ensure JavaScript is enabled
- Check if script.js is loaded
- Verify no JavaScript errors in console

**GitHub Pages not updating:**
- Wait 5-10 minutes for changes to propagate
- Check repository settings for correct branch
- Verify files are in the root directory
